# proposal-agent — 要件定義書

## 概要

ユーザーが指定したテーマについて Web から情報を収集し、構造化された提案資料（Markdown レポート）を自動生成するエージェント。
Chapter 6 の Multi-chain + 評価リトライパターンを採用する。

## 機能要件

### ヒアリング
- ユーザーがテーマ・目的を入力する
- エージェントがヒアリングで要件を明確化する（対話形式）
- ゴールを最適化・具体化する

### 情報収集
- ゴールを検索クエリに分解する
- Tavily API で Web 検索を実行する
- 収集結果を構造化して保持する

### 評価・リトライ
- 収集した情報がゴールを十分にカバーしているか評価する
- 不足している場合は追加クエリで再検索する
- 最大リトライ回数を設ける

### レポート生成
- 収集・評価済みの情報から構造化された Markdown レポートを生成する
- セクション分け、見出し、箇条書きで整理する

## 受け入れ条件

- [ ] テーマ入力 → ヒアリング → ゴール設定の流れが動作する
- [ ] Web 検索で関連情報が収集される
- [ ] 情報不足時にリトライが発生する
- [ ] 構造化された Markdown レポートが生成される
