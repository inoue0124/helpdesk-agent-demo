# langgraph-agent — 要件定義書

## 概要

LangGraph を用いたヘルプデスクエージェントを実装する。質問の分解、検索、回答生成、自己評価（リフレクション）、リトライ、統合の一連のパイプラインを構築する。

## 機能要件

### planner
- 質問を独立したサブタスクに分解する
- 単純な質問は 1 サブタスクのまま通す

### tool_selector
- サブタスクの性質に応じて検索ツールを選択する
  - 具体的な用語 → ES キーワード検索
  - 曖昧・自然言語的 → Qdrant ベクトル検索

### searcher
- 選択されたツールで検索を実行する
- 検索結果をコンテキストとして返す

### answerer
- 検索結果をもとにサブタスクへの回答を生成する
- 検索結果に根拠がない場合はその旨を明示する

### reflector
- 回答品質を自己評価する
  - 検索結果に根拠があるか
  - 質問に過不足なく答えているか
  - 情報が具体的か
- 不十分な場合は別クエリ・別ツールでリトライを指示する
- 最大 3 回リトライ

### integrator
- 全サブタスクの回答を統合して最終回答ドラフトを生成する

### 信頼度算出
- リフレクション結果から信頼度（0.0〜1.0）を算出する
  - 1 回で解決: 1.0
  - リトライで解決: 0.6
  - 未解決: 0.0
- サブタスク全体の平均値を最終信頼度とする

## 非機能要件

- 各ノードの実行を SSE イベントとして外部に通知できる構造にする
- プロンプトは `prompts.py` に集約し、コードとプロンプトを分離する

## 受け入れ条件

- [ ] 質問を入力するとサブタスク分解 → 検索 → 回答 → リフレクションの一連の流れが動作する
- [ ] リフレクションで不十分と判定された場合にリトライが発生する
- [ ] 最大 3 回でリトライが打ち切られる
- [ ] 全サブタスク完了後に統合された最終回答が出力される
- [ ] 信頼度が正しく算出される
- [ ] 各ノードの実行がイベントとして取得できる
