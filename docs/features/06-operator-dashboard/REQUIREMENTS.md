# operator-dashboard — 要件定義書

## 概要

オペレーター向けのダッシュボード画面を構築する。問い合わせ一覧の管理、エージェント処理のリアルタイム可視化、回答の編集・承認を行う。

## 機能要件

### 問い合わせ一覧 (`/operator`)
- 問い合わせをリスト表示する
- ステータスでフィルタできる（未対応 / 対応中 / 完了 / 自動承認）
- 新着問い合わせをリアルタイムまたはポーリングで検知する

### 問い合わせ詳細 (`/operator/[id]`)
- 質問内容、ユーザー名、受付日時を表示する
- エージェント処理の進捗を SSE でリアルタイム表示する
  - 計画（サブタスク分解）
  - サブタスクごとのツール選択 → 検索 → 回答 → リフレクション
  - リトライの可視化（ツール切替、再検索）
  - 統合フェーズ
- 回答ドラフトを編集可能なテキストエリアで表示する
- 信頼度をバッジ（色分け）で表示する
- 承認/却下ボタンで操作する

## 非機能要件

- shadcn/ui + Tailwind CSS でスタイリングする
- SSE 接続は画面遷移時に自動切断する
- レスポンシブ対応（デスクトップ優先）

## 受け入れ条件

- [ ] 問い合わせ一覧がフィルタ付きで表示される
- [ ] 詳細画面でエージェント処理の各ステップがリアルタイムに表示される
- [ ] リトライが発生した場合にその過程が可視化される
- [ ] 回答ドラフトを編集して承認できる
- [ ] 承認後にユーザーに回答が届く
