# helpdesk-agent — 要件定義書

## 概要

ヘルプデスクの問い合わせに対し、ナレッジベースを検索して自動回答するエージェント。
Chapter 4 の Plan→Subtask 分解→Reflection リトライパターンを採用する。

## 機能要件

### エージェント処理
- 質問をサブタスクに分解する（planner）
- サブタスクごとに ES / Qdrant を選択して検索する（tool_selector → searcher）
- 検索結果から回答を生成する（answerer）
- 回答品質を自己評価し、不十分ならリトライする（reflector、最大 3 回）
- サブタスク結果を統合して最終回答を生成する（integrator）

### 信頼度ルーティング
- >= 0.9: 自動承認（ユーザーに即送信 + QA 蓄積）
- 0.5〜0.89: オペレーター確認待ち
- < 0.5: エスカレーション

### ユーザー画面
- お問い合わせフォーム（ユーザー名 + 問い合わせ内容）
- 送信後に受付完了 + 回答到着を表示

### オペレーター画面
- 問い合わせ一覧（ステータスフィルタ）
- エージェント処理のリアルタイム可視化（SSE）
- 回答ドラフトの編集・承認・却下

### フィードバックループ
- 承認された回答を QA ペアとして SQLite + Qdrant に蓄積

## 受け入れ条件

- [ ] 質問 → サブタスク分解 → 検索 → 回答 → リフレクションが動作する
- [ ] リトライが発生し最大 3 回で打ち切られる
- [ ] 信頼度ルーティングが動作する
- [ ] オペレーター画面で進捗がリアルタイム表示される
- [ ] 承認時にフィードバックループが動作する
