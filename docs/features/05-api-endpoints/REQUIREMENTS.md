# api-endpoints — 要件定義書

## 概要

FastAPI の REST / SSE エンドポイントを実装する。問い合わせの受付、一覧・詳細取得、SSE ストリーム、承認/却下 + フィードバックループを提供する。

## 機能要件

### POST /api/inquiries
- 問い合わせを受付し、ステータスを `pending` で作成する
- バックグラウンドでエージェント処理を開始する
- レスポンス: 202 Accepted（id, status, created_at）
- レート制限: 50件/日を超えた場合は 429 エラー

### GET /api/inquiries
- 問い合わせ一覧を取得する（オペレーター用）
- ステータスでフィルタ可能

### GET /api/inquiries/{id}
- 問い合わせ詳細を取得する

### GET /api/inquiries/{id}/stream
- SSE でエージェント処理の進捗をリアルタイム配信する
- 各ノード（plan, subtask, integrating, done）をイベントとして送信する
- エージェント完了後、信頼度に基づいてルーティングする
  - >= 0.9: 自動承認イベント送信
  - 0.5〜0.89: done イベント送信（オペレーター確認待ち）
  - < 0.5: escalated イベント送信

### PATCH /api/inquiries/{id}
- 承認: ステータスを `approved` に変更、最終回答を保存、QA ペアを自動蓄積
- 却下: ステータスを `rejected` に変更

### GET /api/health
- ヘルスチェック（実装済み）

## 非機能要件

- SSE は `sse-starlette` を使用する
- エージェント処理はバックグラウンドタスクとして非同期実行する
- JSON レスポンスは `ensure_ascii=False` で日本語をそのまま返す

## 受け入れ条件

- [ ] 問い合わせの作成→一覧→詳細→更新の CRUD が動作する
- [ ] SSE ストリームでエージェントの各ステップがリアルタイムに配信される
- [ ] 信頼度に基づく自動ルーティングが動作する
- [ ] 承認時にフィードバックループ（QA 蓄積 + Qdrant 登録）が動作する
- [ ] レート制限が機能する
