# portal-frontend — 要件定義書

## 概要

ポータル画面と共通 UI コンポーネントを構築する。ユーザーがエージェントを選択して各デモ画面に遷移できるようにする。

## 機能要件

### ポータルページ (`/`)
- 4 つのエージェントをカード形式で表示する
- 各カードにエージェント名・概要・ユースケース・技術的特徴を表示する
- カードクリックで各エージェントの画面に遷移する

### 共通レイアウト
- ヘッダー: サイトタイトル + ポータルへの戻りリンク
- 各エージェント画面で共通のレイアウトを使用する

### 共通コンポーネント
- AgentProgress: エージェント処理の進捗をステップ表示する
- StreamViewer: SSE イベントをリアルタイム表示する

## 受け入れ条件

- [ ] ポータルページに 4 つのエージェントカードが表示される
- [ ] カードクリックで各エージェント画面に遷移する
- [ ] 全画面で共通レイアウト（ヘッダー + ナビゲーション）が適用される
