# TASKS.md テンプレート

以下のテンプレートに従ってタスクリストを生成すること。

---

```markdown
# <機能名> タスクリスト

対応する設計書: [DESIGN.md](./DESIGN.md)

## Phase 1: <フェーズ名>（例: データモデル・基盤）

同一フェーズ内のタスクは並行実行可能。

- [ ] **<動詞で始まるタスク名>** (規模: S/M/L)
  - ファイル: `path/to/file.py`
  - 内容: 具体的な実装内容を 1-2 行で
  - 依存: なし
  - 完了条件: テスト可能な条件

- [ ] **<タスク名>** (規模: S)
  - ファイル: `path/to/file.py`
  - 内容: ...
  - 依存: なし

## Phase 2: <フェーズ名>（例: バックエンド API）

- [ ] **<タスク名>** (規模: M)
  - ファイル: `path/to/file.py`
  - 内容: ...
  - 依存: Phase 1 完了

## Phase 3: <フェーズ名>（例: フロントエンド）

- [ ] **<タスク名>** (規模: M)
  - ファイル: `frontend/app/...`
  - 内容: ...
  - 依存: Phase 2 の API 完了

## Phase 4: <フェーズ名>（例: 結合・テスト）

- [ ] **<タスク名>** (規模: S)
  - 内容: ...
  - 依存: Phase 2, 3 完了
```

## タスク規模の目安

| 規模 | 目安 |
|---|---|
| S | 単一ファイルの小さな変更。30 分以内 |
| M | 複数ファイルにまたがる変更。1-2 時間 |
| L | 複雑なロジックや多数のファイル変更。半日以上 |
